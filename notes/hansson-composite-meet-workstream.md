# Hansson Composite Meet Workstream

This workstream covers the next belief-base slice only: Hansson 1989
Definitions 3.8 and 3.9, after the existing `BeliefBase.remainder_sets`,
`simple_full_meet_contract`, and `simple_partial_meet_contract` support.

I used the processed Hansson notes and the already-inspected page image for
p.121 as the paper basis for this workstream. I did not reread the full PDF for
this planning pass.

## Paper-Backed Target

- Hansson Definition 3.8: `A || B` is the union of `A perp C` for subsets
  `C` of `B`.
- Hansson Definition 3.8: a subfamily `D` of `A || B` is `B`-covering iff for
  every `b in B` there is some `C subseteq B` such that `b in C` and
  `D intersect (A perp C)` is non-empty.
- Hansson Definition 3.9: composite partial meet contraction is the
  intersection of a selected `B`-covering subfamily of `A || B` when such
  covering is possible.

## Workstream

### WF-1 Composite meet invariant ledger
**Depends on:** none

Record the Definition 3.8/3.9 invariants as the active control surface for this
slice. The target API is `BeliefBase.parallel_sets`,
`BeliefBase.is_covering_selection`, and
`BeliefBase.composite_partial_meet_contract`.

### TDD-1 Parallel family property
**Depends on:** WF-1

Add a red focused or Hypothesis property showing
`parallel_sets(forbidden)` equals the deduplicated union of
`remainder_sets(subset)` over all subsets of `forbidden`.

### FIX-1 Parallel family implementation
**Depends on:** TDD-1

Implement `BeliefBase.parallel_sets(forbidden)`. Preserve source-formula order
inside returned base subsets and reject no valid finite inputs. Keep the method
pure and deterministic.

### TDD-2 Covering selection property
**Depends on:** FIX-1

Add red properties for Hansson's `B`-covering condition:
for every forbidden formula, a covering selection must include at least one
selected member from a remainder family generated by a subset containing that
formula. Also assert invalid selections outside `A || B` are rejected.

### FIX-2 Covering selection implementation
**Depends on:** TDD-2

Implement `BeliefBase.is_covering_selection(forbidden, selected)` plus any
small private helpers needed to enumerate subsets of the forbidden input and
map parallel members back to the remainder families that generated them.

### TDD-3 Composite partial meet property
**Depends on:** FIX-2

Add red tests for Definition 3.9:
`composite_partial_meet_contract(forbidden, selector)` intersects the selected
`B`-covering subfamily of `parallel_sets(forbidden)` when a covering selection
exists. Include examples where composite and simple partial meet can diverge.

### FIX-3 Composite partial meet implementation
**Depends on:** TDD-3

Implement `BeliefBase.composite_partial_meet_contract(forbidden, selector)`.
Validate that selectors choose only members of `A || B`, require covering when
covering is possible, and return the intersection of the selected subfamily as
a new `BeliefBase`.

### DOC-1 Public documentation
**Depends on:** FIX-3

Update the README belief-base section to name `parallel_sets`,
`is_covering_selection`, and `composite_partial_meet_contract`, with one short
example or prose note tying them to Hansson Definitions 3.8/3.9.
